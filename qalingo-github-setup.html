Create the GitHub Repo

Step-by-step. No experience needed. 10 minutes.

⏱ ~10 minutes · Free · No code required
Progress 0 / 5 steps
1
Create a GitHub account
▼
1
Go to github.com and click "Sign up"
2
Enter your email, create a password, and pick a username.
Recommendation: your name or qalingo-app
3
Confirm your email. Choose the Free plan — it's all you need.
💡 Already have GitHub? Skip to Step 2.
GitHub account created and verified
2
Create the QALingo repository
▼
1
Once logged in, click the "+" icon at the top right → "New repository"
2
Fill in the form exactly like this:
Repository name: qalingo
Description: The Duolingo-style app for QA professionals. Learn, certify, get hired.
Visibility: Public ← so anyone can see it
Add README: NO ← we already have one
Add .gitignore: None
License: MIT License
3
Click "Create repository" — the green button at the bottom.
💡 Your repo URL will be: github.com/YOUR-USERNAME/qalingo — save it.
Repository created at github.com/username/qalingo
3
Upload all QALingo files
▼
1
In your new repo, click "uploading an existing file" (the link in the middle of the empty repo page)
2
Download all your QALingo files from Claude and put them in one folder on your computer called qalingo
3
Drag all files at once into the GitHub upload area. These are the files to upload:
📄
README.md
Project documentation
🌐
qalingo-landing.html
Landing page + waitlist
🚀
qalingo-onboarding.html
Onboarding flow
🎮
qalingo-lesson-app.html
ISTQB lesson experience
🎯
qalingo-istqb-simulator.html
40-question exam simulator
💼
qalingo-interview-prep.html
AI interview practice
🏆
qalingo-gamification.html
XP, badges, leaderboard
📋
qalingo-architecture.html
Product blueprint
🔧
qalingo-backend-guide.html
Supabase + Bubble guide
4
Scroll down to "Commit changes". In the message box write: Initial commit — QALingo MVP
5
Click "Commit changes" — the green button. Done.
All 9 files uploaded to the repo
First commit message written and saved
4
Enable GitHub Pages — free hosting
▼
1
In your repo, click Settings (top menu, last tab)
2
Scroll down to Pages in the left sidebar
3
Under Source → select Deploy from a branch → Branch: main → Folder: / (root)
4
Click Save. Wait 2 minutes. Your landing page will be live at:
Landing URL: https://YOUR-USERNAME.github.io/qalingo/qalingo-landing.html
Lesson app: https://YOUR-USERNAME.github.io/qalingo/qalingo-lesson-app.html
Exam sim: https://YOUR-USERNAME.github.io/qalingo/qalingo-istqb-simulator.html
💡 To make the landing page load at the root URL, rename qalingo-landing.html to index.html before uploading. Then it'll be live at username.github.io/qalingo directly.
GitHub Pages enabledSettings → Pages → main branch
Landing page live and accessible via URL
5
Polish the repo — topics + description
▼
1
Back on the repo main page, click the ⚙️ gear icon next to "About" (top right of the file list)
2
Set the Website field to your GitHub Pages URL
3
Add these Topics — they help people find the project:
qa-testing istqb software-testing duolingo
gamification interview-prep learning-platform qalingo
4
Click Save changes. Your repo now looks professional.
Website URL set in repo About section
Topics added to the repo
