Fase 1 de 4
Configurar Supabase
Tu base de datos. Esto es donde vive toda la información de los usuarios de QALingo.
1.1
Crear cuenta y proyecto
~10 min
1
Ir a supabase.com y crear cuenta gratis
Usá tu Gmail o GitHub para registrarte. No necesitas tarjeta de crédito.
2
Crear un nuevo proyecto
Click en "New Project". Nombre: qalingo. Región: la más cercana a vos (US East o EU). Contraseña: guardala en algún lugar seguro.
3
Guardar tus credenciales
En Settings → API vas a encontrar dos cosas importantes. Guardalas:
GUARDAR ESTO
Project URL: https://xxxxx.supabase.co
anon public key: eyJhbGciOiJIUzI1NiIs...
# Vas a necesitar estos dos valores en Bubble
1.2
Crear las tablas de la base de datos
~30 min
En Supabase, ir a Table Editor → New Table. Crear estas 4 tablas exactamente con estas columnas:
📋 Tabla: users
ColumnaTipoNotas
id PKuuidAuto-generado por Supabase Auth
emailtextÚnico, no nulo
display_nametextNombre visible en la app
levelint4Default: 1
xp_totalint4Default: 0
streak_currentint4Default: 0
streak_longestint4Default: 0
heartsint4Default: 5, Max: 5

remind_timetextmorning/lunch/evening
last_activetimestamptzPara calcular streaks
created_attimestamptzAuto: now()

ColumnaTipoNotas

🏅 Tabla: user_badges
ColumnaTipoNotas

💡
Tip: Al crear cada tabla, Supabase pide si querés habilitar "Row Level Security (RLS)". Di que SÍ siempre. Luego en Bubble vas a configurar qué puede ver cada usuario.
1.3
Activar Autenticación
~5 min
1
Ir a Authentication → Providers
Activar Email (ya viene activado). Opcionalmente activar Google para login con Gmail — muy recomendado, reduce fricción.
2
Configurar el email de confirmación
En Authentication → Email Templates, personalizar el asunto con la voz de Q:
EMAIL TEMPLATE
Subject: 🔍 Confirm your QALingo account, Watson
Body: "The investigation begins once you confirm your identity."
# Agregar el link de confirmación como siempre
Fase 2 de 4
Configurar Bubble.io
Acá construís la app visualmente y la conectás a Supabase como backend.
2.1
Crear cuenta y nuevo proyecto
~10 min
1
Registrarse en bubble.io
Plan gratuito alcanza para el MVP. Vas a tener una URL como qalingo.bubbleapps.io hasta que conectes un dominio propio.
2
Crear nuevo proyecto — "QALingo"
Elegir Blank app. No usar templates — nuestros diseños ya están hechos y son mejores.
3
Instalar el plugin de Supabase
En Plugins → Add plugins → buscar "Supabase". Instalarlo. Luego en la configuración del plugin pegar:
BUBBLE PLUGIN CONFIG
Supabase URL: https://xxxxx.supabase.co
Supabase Anon Key: eyJhbGciOiJIUzI1NiIs...
# Los dos valores que guardaste en el Paso 1.1
2.2
Construir las páginas principales
~3-5 días
Estas son las páginas que necesitás crear en Bubble. El diseño visual ya lo tenés en los archivos HTML — usá esos como referencia exacta.
1
index — Login / Registro
Formulario de email + contraseña. Botón "Continue with Google". Al registrarse exitosamente → redirigir a /onboarding.
2
onboarding — El flow que ya construimos
Al completar, guardar en Supabase: goal, experience, remind_time en la tabla users. Luego redirigir a /dashboard.
3
dashboard — El mapa de lecciones
Leer de Supabase: nivel del usuario, XP, streak. Mostrar el path de aprendizaje con los módulos disponibles según su plan (free/pro).
4
lesson — La experiencia de lección
Al completar cada ejercicio: +XP en users, insertar fila en lesson_progress, insertar en daily_activity, calcular si se gana badge.
5
profile — Gamificación y badges
Leer de Supabase: user_badges (badges ganados), daily_activity (historial XP), users (streak, nivel). Todo lo que ya está en el dashboard de gamificación.
🔍
La lógica del streak en Bubble: Cada vez que el usuario completa una lección, un Workflow revisa si ya hay una fila en daily_activity para HOY. Si no hay → crear una + sumar 1 al streak_current en users. Si ya hay → no hacer nada. A medianoche, un scheduled workflow revisa quién no tuvo actividad ese día y resetea streak a 0.
2.3
Lógica de Badges automática
~2 horas
En Bubble, crear un Backend Workflow que se ejecuta después de cada lección completada. Revisa estas condiciones en orden:
# WORKFLOW: check_badges (corre después de completar lección) IF lesson_progress.count WHERE user = current_user = 1 THEN award_badge('first_bug') // Primera lección IF users.streak_current = 7 THEN award_badge('streak7') // 7-day streak IF users.streak_current = 14 THEN award_badge('streak14') // 14-day streak IF users.xp_total >= 1000 THEN award_badge('watson') // 1000 XP total # award_badge = INSERT en user_badges + trigger email en Loops
⚠️
Importante: Antes de insertar el badge, siempre revisar que el usuario no lo tenga ya en user_badges. Así evitás duplicados.
Fase 3 de 4
Configurar Loops.so — Los emails
Los emails de streak son el mecanismo de retención más importante. Sin ellos, la mitad de los usuarios no vuelve al día 3.
3.1
Crear los 4 emails clave
~2 horas
1
Email de Bienvenida — se envía al registrarse
Asunto: 🔍 The investigation begins, Watson

Cuerpo:
Welcome to QALingo.

Your path is set. Your first lesson waits.
Q is watching. Don't disappoint the detective.

[Start my first lesson →] // Button → tu app URL
2
Recordatorio diario — se envía a las 8pm si no hizo lección
Asunto: 🔥 Your {streak}-day streak is at risk, Watson

{first_name}, Q has been waiting all day.
Your streak: {streak} days 🔥
Time left today: ~3 hours

[Keep my streak alive →]
3
Streak perdida — se envía cuando streak llega a 0
Asunto: 💀 The bug got away, Watson

Your {streak}-day streak is gone.
Even great detectives lose cases.
The question is: do they give up?

[Start a new streak →]
4
Badge ganado — se envía cada vez que se gana un badge
Asunto: 🏅 Badge unlocked: {badge_name}

QAlemental, {first_name}.
You just earned: {badge_emoji} {badge_name}
+{badge_xp} XP added to your profile.

[Share on LinkedIn →] // LinkedIn share URL
🔍
Cómo conectar Bubble → Loops: En Loops, ir a API → copiar el API Key. En Bubble, en cada Workflow donde querés enviar un email, usar el action "API Connector" y llamar al endpoint de Loops con el email del usuario y las variables del template.
Fase 4 de 4
Landing page + Waitlist en Framer
La landing que ya construimos publicada en minutos. El form de waitlist conectado a Loops.
4.1
Publicar la landing page
~1 hora
1
Opción A (Más fácil): GitHub Pages — Gratis
Crear cuenta en github.com. Crear un repositorio llamado qalingo-landing. Subir el archivo qalingo-landing.html (renombrarlo a index.html). Activar GitHub Pages en Settings → Pages. Tu URL será: tuusuario.github.io/qalingo-landing
2
Opción B (Más profesional): Netlify — Gratis
Ir a netlify.com. Arrastrar el archivo index.html al área de deploy. En 30 segundos tenés una URL pública. Podés conectar tu dominio propio gratis.
3
Conectar el form de waitlist a Loops
En el archivo HTML de la landing, buscar la función handleSignup() y agregar esta llamada a la API de Loops:
// En handleSignup() — después de validar el email fetch('https://app.loops.so/api/v1/contacts/create', { method: 'POST', headers: { 'Authorization': 'Bearer TU_LOOPS_API_KEY', 'Content-Type': 'application/json' }, body: JSON.stringify({ email: email, source: 'waitlist', mailingLists: { 'qalingo_waitlist': true } }) })
